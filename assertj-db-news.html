AssertJ-DB 1.1.0 :

Release date : 2016-04-14 (dependency declaration)

Javadoc : for versions 1.1.x

Improvements :
  • Add from(Calendar) and now() methods to DateValue, TimeValue and DateTimeValue
  • issue #9 : Make a way to view the data of a Table, of a Request or of Changes with Displaying
  • issue #15 : Add support of UUID columns type (Otoniel Isidoro)
  • Add isOfClass(Class) assertion method for value and column
  • issue #18 : Add isCloseTo(...) methods for Number, DateValue, TimeValue and DateTimeValue
  • issue #19 : Add the support of BLOB and CLOB
  • issue #22 : Add isEqualTo(Object) for the values. Add hasValues(Object...) for the columns. Add hasValues(Object) and hasValues(Object, Object) for the columns of the changes
  • issue #25 : Add containsValues(...) methods for the columns
  • issue #29 : Add hasNumberOfXXXGreaterThan(...), hasNumberOfXXXLessThan(...), hasNumberOfXXXGreaterThanOrEqualTo(...) and hasNumberOfXXXLessThanOrEqualTo(...) methods for the rows, columns, changes and modified columns
  • issue #34 : Enhance exception message when column does not exist
Bug fixes :
  • issue #21 : Add possibility to pass a reference containing null value like parameter to isEqualTo() and isNotEqualTo()
  • issue #23 : Fix the support of Number (bug when the mapping is a Double instance)
  • issue #31 : Fix the detection of primary key (cause by letter case) in some DBMS
  • issue #32 : Fix the SQL request for DBMS with letter case different of upper case in the name of the DB elements
