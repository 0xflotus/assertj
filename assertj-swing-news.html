AssertJ Swing latest news

AssertJ Swing 2.3.0

Release date : 2017-11-15

Version 2.3.0 is a maintenance release which updates AssertJ Core dependency and should increase stability.

AssertJ Swing 3.8.0

Release date : 2017-08-09

Version 3.8.0 contains new features:

  • Made AbstractComponentFixture.robot() public
  • Allow multiple screen configurations in point-on-screen-check in AWT.

and some fixes:

  • JavaDoc references fixed.

AssertJ Swing 3.7.0

Release date : 2017-07-12

Version 3.7.0 contains new features:

  • #59 Add drop method to AbstractComponentFixture
  • #122 Override equals and hashCode in AbstractComponentFixture
  • #198 Log information about keystroke mapping and the provider lookup
  • JIDE 3.6.21 and assertj-core 3.8.0

and some fixes:

  • Calculation of the center point of a JList cell
  • Fixed TextNG before and after methods when running with groups

AssertJ Swing 3.6.0

Release date : 2017-05-16

Version 3.6.0 contains a new feature:

  • #203 JTabbedPane: Expose ToolTipText and Enabled
  • JIDE 3.6.20 and assertj-core 3.7.0

and a bug fix:

  • #169 Allow opening popup menu on disabled table cells

AssertJ Swing 3.5.0

Release date : 2016-12-14

Version 3.5.0 contains a new feature:

  • #181 FrameFixture should expose `requireTitle`
  • JIDE 3.6.16 and assertj-core 3.5.2

and bug fixes:

  • #194 Exception within Settings

AssertJ Swing 3.4.0

Release date : 2016-07-04

Version 3.4.0 contains a new feature:

  • #183 Drop a List or Tree item item above or below a specified item
  • #185 Finnish keymap support
  • JIDE 3.6.14 and assertj-core 3.5.1

and bug fixes:

  • #184 findActivePopup cascading popups not working correctly
  • #187 RobotEventGenerator uses AWT.translate() outside the EDT
  • Wait for idle not waiting as JavaDoc of Object#wait suggests

AssertJ Swing 3.3.0

Release date : 2016-04-25

Version 3.3.0 contains new features:

  • #56 Add #require(Not)Armed to AbstractTwoStateButtonFixture
  • #182 Ability to unselect rows on a JTableFixture
  • Possibility to invoke GuiActionRunner with lambda expressions.
  • Possibility to press mouse/keys while running a Runnable passed via lambda expressions. See Robot's API for new methods that take an additional Runnable as argument.

AssertJ Swing 3.2.0

Release date : 2016-02-29

Version 3.2.0 contains new features:

  • #3 Allow screenshots to remain on disk
  • #119 Mouse button configurable for Drag and Drop
    see Configuration for basic and Settings.java for further details.

and a couple of bug fixes:

  • #10 Passing true values for isSelected and cellHasFocus in BasicJListCellReader
  • #22 JMenuItemDriver doClick() should check for 'inMenuBar()' before use special OS X logic
  • #24 JTableHeaderLocation.columnName() doesn't work for reordered columns
  • #180 StackOverflowError on BasicRobot.findOuterPopupMenu()

AssertJ Swing 3.1.1

Release date : 2016-02-15

Version 3.1.1 contains:

  • #120 JTabbedPaneFixture missing require-method for selected tab
  • #169 Clicking on disabled Swing components

  • #173 BasicJTableCellWriter unable to enter an empty string

AssertJ Swing 3.0.0

Release date : 2015-10-05

Version 3.0.0 is the first Java 8 release and uses new dependencies:

  • org.testng:testng 6.9.6 (5.7)
  • org.slf4j:slf4j-api 1.7.12 (1.7.10)
  • commons-codec:commons-codec 1.10 (1.3)
  • org.apache.ant:ant-junit 1.9.6 (1.7.0)
  • com.google.code.findbugs:jsr305 3.0.0 (1.3.9)
These won't have an effect on you but it might be interesting to know :)

AssertJ Swing 1.1.0

Release date : 2015-02-16

Version 1.1.0 is the first AssertJ Swing release after a long period and it comes with a new feature:

Improvements
  • Convenient methods for using Containers with own Robot: 
    Containers.frameFixtureFor(robot(), panel);
Containers.showInFrame(robot(), panel);
  • JIDE-support is added by the new module assertj-swing-jide

AssertJ Swing 1.0.0

Release date : 2014-07-22

Version 1.0.0 is the first AssertJ Swing release after the fork from FEST-Swing (version: Mar 3, 2013), here are the changes compared to FEST-Swing:

Improvements
  • Ensure timeouts
  • Make Swing module work with latest AssertJ core
  • Reduce published dependencies
  • Stabilise AssertJ Swing under Java 7
